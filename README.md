## О проекте

Конфигурация охватывает полный цикл работы дневного стационара: приём пациентов, размещение по палатам и отделениям, учёт оказанных медицинских услуг, расход медикаментов и питания, выписка и расчёт стоимости пребывания.

Исходники выгружены в XML-формате штатным механизмом платформы (**Конфигурация → Выгрузить конфигурацию в файлы...**) и версионируются через git. Формат позволяет вести историю изменений и работает на базовых версиях платформы без хранилища конфигурации.

---

## Возможности

- Оформление приёма и выписки пациентов дневного стационара
- Размещение пациентов по палатам и отделениям, отслеживание сроков пребывания
- Учёт оказанных медицинских услуг и расчёт стоимости пребывания
- Учёт расхода и списания медикаментов и питания по пациентам
- Отчёт по загрузке палат/отделений и доходам учреждения

## Состав конфигурации

### Справочники

| Справочник | Назначение |
|---|---|
| `Пациенты` | Карточки пациентов |
| `Пол` | Справочник пола пациента |
| `Палаты` | Палаты стационара |
| `ОтделениеСтационара` | Отделения стационара |
| `МедицинскиеУслуги` | Перечень оказываемых медицинских услуг |
| `ЛекарственныеПрепараты` | Номенклатура лекарственных препаратов |
| `Питание` | Номенклатура рационов/позиций питания |
| `Сотрудники` | Карточки сотрудников учреждения |

### Документы

| Документ | Назначение |
|---|---|
| `ПриемВДневнойСтационар` | Оформление приёма пациента в дневной стационар |
| `ПребываниеПациента` | Учёт пребывания пациента (размещение, сроки) |
| `ВыпискаПациента` | Оформление выписки пациента |
| `РасходМедикаментов` | Списание/расход медикаментов на пациента |
| `РасходПитания` | Списание/расход питания на пациента |
| `СписаниеМедикаментовИПитания` | Общее списание медикаментов и питания со склада |

### Регистры и расчёты

| Объект | Назначение |
|---|---|
| `РасчетСтоимостиПребывания` | Расчёт стоимости пребывания пациента в стационаре |

### Отчёты

| Отчёт | Назначение |
|---|---|
| `ОтчетПоЗагрузкеИДоходам` | Отчёт по загрузке палат/отделений и доходам учреждения |

## Разработка

### Обновление репозитория после изменений конфигурации

1. Откройте базу в режиме **Конфигуратор**.
2. Выполните **Конфигурация → Выгрузить конфигурацию в файлы...**, указав папку репозитория — изменятся только реально изменённые файлы.
3. Зафиксируйте изменения в git:

```bash
git add .
git commit -m "Описание изменений"
git push
```

### Развёртывание конфигурации из репозитория

1. Создайте новую информационную базу в 1С (или используйте существующую).
2. В Конфигураторе выполните **Конфигурация → Загрузить конфигурацию из файлов...**, указав путь к папке репозитория.
3. Обновите конфигурацию базы данных (**F7**).
